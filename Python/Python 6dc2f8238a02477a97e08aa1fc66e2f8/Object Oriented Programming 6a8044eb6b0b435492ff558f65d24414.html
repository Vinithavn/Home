Object Oriented Programming

Classes

Class Vehicle:
	no_of_wheels = 4 
	def stop(self):
		print("Stopping the vehicle")

Here Vehicle is the name of our class. no_of_wheels is an attribute of the class, and stop() function is a method specific to this class

Objects

car = Vehicle()
print(car.no_of_wheels)
print(car.stop())

Here we are creating an object for the class Car. We can access the class attributes and methods for this object using the dot operator. In the above example car.no_of_wheels will output the value 4 and car.stop() will print ‚ÄúStopping the vehicle‚Äù

Constructors in a class

Constructors in a class is used for initializing the variables in a class. This is defined using an init function. Init function is automatically called whenever we create a new object. Consider the below example. Here whenever an object of class Vehicle is created, the number of wheels attribute is initialized and is set to 4. 

Class Vehicle:
	def __init__(self)
		self.no_of_wheels = 4 
	def stop(self):
		print("Stopping the vehicle")

Why OOPs

  • Procedural programming - Without OOPs
  • Python is an OOP Language
  • We can take chunks of code and modularize it
  • Each of the modules can be reused whenever needed
  • We cant use procedural programming for a large or complex program
  • In those cases, we can make use of OOPs
  • How to use OOP
    • By creating classes and objects
    • A class is a blueprint that has some attributes and methods
    • Objects are instances of the class that are used in the program
    • Therefore an object will have attributes(variables) and methods(functions)

Difference between class and module

  • A module can be considered a python script and it can have zero or more classes
  • To import a variable or function from a module, we can just use
    • Consider a module called another_module.py in a project
    another_variable = 12
    • Now from main.py inside the same project, we can access the module as:
    import another_module 
print(another_module.another_variable)
    • If another_module.py is like this
      another_variable = 12
class Myclass:
	def __init__():
		class_variable = 13

      Then from main.py to access these two variables, the process is different

      import another_module 
print(another_module.another_variable)

from another_module import Myclass 
class_obj = Myclass()
print(class_obj.class_variable)
      • Here another_module.py is a module and Myclass is the class that is defined inside that module
